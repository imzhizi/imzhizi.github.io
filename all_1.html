安装说明

Homebrew 是 macOS 下的一个非常好用的包管理工具, caskroom 则是基于 Homebrew 构建的一个强大的应用程序管理器.

API 鉴权

鉴权

做 API 服务必须要鉴权, 根据鉴权结果来判断是否允许用户访问, 主流方法有三种:

  • 用户名和密码鉴权，使用 session 保存用户鉴权结果
  • 自行采用 Token 进行鉴权
  • 使用 OAuth 进行鉴权（其实 OAuth 也是一种基于 Token 的鉴权，只是没有规定Token的生成方式）

Session - Cookie

Session 是一种久远的技术, 首先它只适用于 web 项目, 它跟别的技术完全不同. Session 一般由 Web 容器实现, 比如说 Tomcat 实现了 JSession, Tomcat 会为每一个连接开辟一块内存用来存储对应的相关信息, 在浏览器端通过 cookie 来维护该连接(cookie 中使用 JSessionID 来进行匹配), 整体被抽象为 session, 一旦登录成功, 服务器可以在 session 中保存某些认证信息(比如说用户名), 这里的点在于, 只要 session 中存在用户名, 一定是服务器放进去的, 因为维护 session 的 cookie 是加密的.

弊端:

使用 session 方案某些场景下是无法满足需求的, 尤其是在多服务器的情况下, 此时 session 并不在服务器间共享, 所以鉴权结果无法取出.

Suppose you have n versions [1, 2, ..., n] and you want to find out the first bad one, which causes all the following ones to be bad.
You are given an API bool isBadVersion(version) which will return whether version is bad. Implement a function to find the first bad version. You should minimize the number of calls to the API.

Example:
Given n = 5, and version = 4 is the first bad version.

call isBadVersion(5) -> true
call isBadVersion(4) -> true

Then 4 is the first bad version.
