53 Maximum Subarray

2018/4/25 21:34 下午 posted in  算法

题目描述

Given an integer array nums, find the contiguous subarray (containing at least one number)
which has the largest sum and return its sum.

Example:
Input: [-2,1,-3,4,-1,2,1,-5,4],
Output: 6
Explanation: [4,-1,2,1] has the largest sum = 6.

题目总结

  • 这道题，一拿到手脑中想到的就是穷举法，操作就是把整个数组所有的子串遍历一遍，求出它们的子串和。但非常遗憾，穷举法需要三重循环，O(n3)的时间复杂度让我成功的超时了；
  • 那要怎么办呢？我发现这道题似乎可以用动态规划来做，但我从来没有接触过动态规划。经过看书，我首先得知动态规划一个重要原则就是解决小实例，存储其结果，形成可供参考的数组表；
  • 于是我首先发现的就是对于从0到j的和，减去从0到i-1的和，就是i到j的和，这样就不用每次累加计算了，于是一个O(n2)时间复杂度的算法就诞生了；

动态规划

进一步学习了解这道题之后，我学到了一个新的缩小问题域的方法：

  • 令result[j]表示以位置 j 为终点的所有子区间中和最大的一个；
  • 子问题: 如 j 为终点的最大子区间包含了位置j-1,则以j-1为终点的最大子区间必然包括在其中；
  • 如果令result[j-1] >0, 那么显然result[j] = result[j-1] + nums[j]，因为之前一个是最大的，那么加上nums[j]即可，因为必须包含nums[j]；
  • 如果result[j-1]<=0,那么result[j] = nums[j]，因为既然最大，前面的负数必然不能使你更大；

代码实现

详解见注释

/**
 * created by zhizi on 2018/4/24
 */
public class A53_MaximumSubarray {
    /**
     * 函数描述：
     * 尝试使用动态规划来写这道题，将问题域进行转化；
     * 使用数组sum[j]，j表示结尾为j的子列的最大和，注意⚠️：是️以j为结尾，所以这个结果中一定包含了nums[j]；
     * 如果j结尾的最的子列长度大于1，那么sum[j]=sum[j-1]+num[j]；
     * 子串必定以数据中的某个数字结尾，因此最大和也就是sum[]中的最大值；
     * 运行时长：15 ms
     *
     * 总结：
     * 这道题学到很多，亲手把一个三重循环优化到二重再到一重，还少了一次排序，可见有很多操作都是不必要的；
     * 所有保存结果找最大值都可以参考这道题，边计算边寻找最大值；
     * 另一方面如果存在递推，一个结果可能被多次计算，那就可以通过数表来查找数据进而减少计算。
     */
    public static int maxSubArray3(int[] nums) {
        //最终版，把存放结果的数组变成lmax
        int max = nums[0];
        int lmax= nums[0];
        for (int i = 1; i < nums.length; i++) {
            if (lmax > 0) {
                lmax = lmax + nums[i];
            } else {
                lmax = nums[i];
            }
            max = Math.max(max, lmax);
        }
        return max;

        //原始版，使用数组保存结果
//        int[] result = new int[nums.length];

//            if (result[i - 1] > 0) {

//            } else {
//                result[i] = nums[i];
//            }
//            max = Math.max(max, result[i]);
//        }
//        return max;
    }

    /**
     * 函数描述：
     * 最初只想到穷举法，通过三重循环直接把所有可能的结果累加计算出来，然后通过排序找到最大值；
     * 经过优化成为二重循环，因为从i到n的和，都可以通过1到n的和减去1到i-1的和得出，可以节约一重循环；
     * 而通过排序寻找最大值也要一次O(nlogn)，其实可以边计算边寻找最大值；
     * result我本来是n*n的二维数组，将所有的结果保存起来，结果会空间超支，后来发现可以只用一个数字。
     * 运行时长：198 ms
     *
     * 总结：见上方
     */
    public static int maxSubArray2(int[] nums) {
        int max = nums[0];
        int temp = 0;
        for (int i = 0; i < nums.length; i++) {
            for (int j = i; j < nums.length; j++) {
                if (i == j)
                    temp = nums[i];
                else
                    temp = temp + nums[j];
                max = Math.max(max, temp);//边计算边寻找最大值
            }
        }
        return max;
    }

    public static void main(String[] args) {
        System.out.println(maxSubArray2(new int[]{-2,1,-3,4,-1,2,1,-5,4}));
    }
}
